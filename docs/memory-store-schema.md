# Memory Store Schema

The NE-Memory engine stores all entries in `.memory/ne/memory.db` (SQLite + FTS5).

## Schema

The `memories` table has the following columns:

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT (PK) | UUID v4 — auto-generated by `add_memory()`. Unique per entry. |
| `text` | TEXT | The free-text memory content. Indexed by FTS5. Max ~50KB. |
| `source` | TEXT | Origin label: `manual`, `task_learning`, `code_preference`, `system`, `test`. |
| `timestamp` | TEXT | ISO 8601 UTC timestamp of when the entry was created. |
| `metadata` | TEXT (JSON) | Arbitrary JSON metadata. Can contain `topic`, `type`, `confidence`, or any custom fields. Default: `{}`. |
| `vector` | BLOB | Optional embedding vector for semantic search (float32 array). Null if not computed. |

An FTS5 virtual table `memories_fts` provides BM25 full-text search over `text`, `source`, and `metadata`.

## Metadata Conventions

The system uses these metadata conventions:

- `metadata.type = "code_preferences"` — behavioral/process fixes
- `metadata.type = "task_learning"` — factual domain learnings (default for most entries)
- `metadata.type = "user_preference"` — user-stated preferences
- `metadata.topic` — domain topic for grouping (e.g. "spaced-repetition", "rust", "testing")
- `metadata.confidence` — confidence score (0.0-1.0) used during consolidation to prefer higher-confidence entries

## Backing Files

`.memory/ne/` contains:

| File | Purpose |
|------|---------|
| `memory.db` | SQLite database with memories + FTS5 index + triggers |
| `memories.json.imported` | Legacy JSON import (renamed after migration) |
| `data/synonyms.json` | Synonym map for query expansion |
| `data/synonyms.json` | Synonym expansion map for BM25 query broadening |

## BM25 Scoring

The BM25 engine computes scores at query time using:

- **k1** = 1.2 (term saturation factor)
- **b** = 0.75 (length normalization)
- **Tokenization**: lowercase, alphanumeric-only, stopword-filtered
- **Synonym expansion**: entries in `synonyms.json` are expanded at query time
- **Fuzzy matching**: Levenshtein distance via difflib (default threshold 0.85)

No LLM calls are made for any memory operation.
