# Memory Store Schema

The NE-Memory engine stores all entries in `.memory/ne/memories.json`.

## Schema

```json
[
  {
    "id": "uuid-string",
    "text": "The actual memory content as free-text",
    "source": "manual | task_learning | code_preference | system | test | <custom>",
    "timestamp": "2026-07-16T22:34:35Z",
    "metadata": {
      "topic": "optional-topic-tag",
      "type": "optional-type-tag",
      "confidence": 0.0-1.0,
      "...": "any additional key-value pairs"
    }
  }
]
```

## Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | UUID v4 — auto-generated by `add_memory()`. Unique per entry. |
| `text` | string | yes | The free-text memory content. Tokenized for BM25 indexing. Max ~50KB. |
| `source` | string | yes | Origin label: `manual` (user-entered), `task_learning` (agent learned), `code_preference` (behavioral rule), `system` (auto-generated), `test` (test harness). |
| `timestamp` | string | yes | ISO 8601 UTC timestamp of when the entry was created. |
| `metadata` | object | yes | Arbitrary JSON metadata. Can contain `topic`, `type`, `confidence`, or any custom fields. Default: `{}`. |

## Metadata Conventions

The agent-memory-mcp system uses these metadata conventions:

- `metadata.type = "code_preferences"` — behavioral/process fixes (filtered separately in search)
- `metadata.type = "task_learning"` — factual domain learnings (default for most entries)
- `metadata.type = "user_preference"` — user-stated preferences
- `metadata.topic` — domain topic for grouping (e.g. "spaced-repetition", "rust", "testing")
- `metadata.confidence` — confidence score (0.0-1.0) used during consolidation to prefer higher-confidence entries

## Index File

`.memory/ne/` also contains:

| File | Purpose |
|------|---------|
| `memories.json` | All memory entries (above schema) |
| `data/synonyms.json` | Synonym expansion map for BM25 query broadening |

## BM25 Scoring

The BM25 engine computes scores at query time using:

- **k1** = 1.2 (term saturation factor)
- **b** = 0.75 (length normalization)
- **Tokenization**: lowercase, alphanumeric-only, stopword-filtered
- **Synonym expansion**: entries in `synonyms.json` are expanded at query time
- **Fuzzy matching**: Levenshtein distance via difflib (default threshold 0.85)

No LLM calls are made for any memory operation.
